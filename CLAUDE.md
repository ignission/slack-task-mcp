# CLAUDE.md - Slack Task MCP Server

このプロジェクトはClaude Code用のSlackタスク管理MCPサーバーです。

## 概要

ADHDの特性を持つユーザー向けに、Slackの依頼を分析・分解し、小さなステップで達成感を得られるようにするツールです。

## 利用可能なMCPツール

### get_slack_thread

SlackスレッドのURLからメッセージを取得します。

```
使用例: このスレッドを読んで https://xxx.slack.com/archives/C12345678/p1234567890
```

### save_task

タスクを保存します。分析・分解後に使用してください。

### list_tasks

保存されているタスクの一覧を表示します。

### complete_step

タスクのステップを完了にします。

## タスク分解のルール

Slackの依頼を分解する際は、以下のルールに従ってください：

### 1. 目的の明確化

- この依頼の最終ゴールは何か？
- 何を作る/する必要があるか？
- 誰のため？誰と連携？

### 2. 不明点の洗い出し

- 依頼文から読み取れない情報をリストアップ
- 「たぶんこうだろう」で進めると手戻りになる点を特定
- 不明点がある場合は、確認メッセージ案を提示

### 3. タスク分解

- 1ステップは5分以内で完了できる粒度に分解
- 最初のステップは最も簡単で、すぐ終わるものにする
- 各ステップは「やった感」が得られる区切りにする
- 曖昧な作業は具体的な動作に変換する
  - ❌「資料を作る」
  - ✅「Googleドキュメントを開いてタイトルを書く」

### 4. 優先度の判定

- 🔴 高: 他の人をブロックしている、期限が近い
- 🟡 中: 今日〜今週中にやるべき
- 🟢 低: いつでもいい

## タスクタイプ別のTips

### 報告系タスク

```
基本構成:
1. 結論（何を伝えたいか）
2. 根拠（なぜそう言えるか）
3. 次のアクション（何をしてほしいか）
```

### 調査系タスク

```
基本構成:
1. 調査の目的・背景を確認
2. 情報源を特定
3. 情報を収集
4. 整理・要約
5. 報告
```

### 作成系タスク

```
基本構成:
1. 要件・仕様を確認
2. 構成を決める
3. 下書きを作る
4. レビュー・修正
5. 完成・提出
```

## ワークフロー例

```
ユーザー: このスレッドを分析して https://xxx.slack.com/...

Claude: [get_slack_thread でスレッド取得]

Claude:
## 依頼の分析

### 把握した内容
- 目的: 〇〇
- 成果物: 〇〇
- 期限: 〇〇

### 不明点
- ❓ 〇〇について確認が必要

### 確認メッセージ案
「〇〇について確認させてください。〇〇でしょうか？」

---

ユーザー: 確認した。〇〇だった。分解して保存して。

Claude: [save_task でタスク保存]

✅ タスクを保存しました

### 週次レポート作成
1. ☐ 先週のSlackを5分で振り返る (5分)
2. ☐ 箇条書きで3つ書き出す (5分)
3. ☐ 各項目を1文ずつ膨らませる (5分)
4. ☐ 数字を入れる (3分)
5. ☐ 見直して送信 (2分)

📌 まず1から始めましょう。5分で終わります。

---

ユーザー: 1完了

Claude: [complete_step で完了]

✅ ステップ 1 を完了しました！

~~先週のSlackを5分で振り返る~~

📌 次のステップ: 2. 箇条書きで3つ書き出す (5分)
```

## データ保存先

```
~/.slack-task-mcp/tasks.json
```

## 注意事項

- Slack User Token (xoxp-...) が環境変数 `SLACK_USER_TOKEN` に設定されている必要があります
- User Tokenを使用するため、Botをチャンネルに追加する必要はありません
- プライベートチャンネルも、あなたが参加していれば読み取り可能です
