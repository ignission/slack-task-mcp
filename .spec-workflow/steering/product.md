# Product Document

## プロダクト名

Slack Task MCP Server

## ビジョン

Slackのメンションから着手までの認知的摩擦をゼロにする。

## 解決する課題

Slackでメンションがたまると、どこから手を付けていいか迷い、難しい依頼は諦めてしまう問題を解決する。

### ペインの詳細

```
メンション来た
    ↓
「何求められてる？」が曖昧で固まる → 目的の明確化が必要
    ↓
「何聞けばいい？」がわからない   → 不明点の洗い出しが必要
    ↓
「どう返せばいい？」で時間かかる → 返信の構造化が必要
    ↓
「次何する？」で迷う            → ネクストアクション提示が必要
```

## ターゲットユーザー

**ADHDの特性を持つワーカー**

- Slackのメンションがたまると、どこから手を付けていいか迷う
- 難しい依頼が来ると、何を聞けばいいかわからず固まる
- 返信を書くのに時間がかかる、伝わるか不安になる
- タスクを分解しても、途中で集中が切れて諦めてしまう

## コアバリュー

1. **認知負荷の軽減**: 依頼の目的を明確化し、次にやるべきことを提示
2. **行動の着火**: 5分以内で完了できる小さなステップに分解
3. **文脈の外部化**: Slackスレッドを記憶に頼らず参照可能に

## 主要機能

| 機能 | 説明 | 状態 |
|------|------|------|
| スレッド取得 | SlackスレッドのURLからメッセージを取得 | ✅ |
| 依頼分析 | 目的・不明点・確認メッセージ案・優先度を生成 | ✅ |
| 返信添削 | 結論→根拠→アクションの構造に整理 | ✅ |
| タスク保存 | ステップ付きタスクとして永続化 | ✅ |
| タスク一覧 | アクティブなタスクを表示 | ✅ |
| タスク検索 | キーワード・日付・ステータスで検索（アーカイブ含む） | ✅ |
| ステップ完了 | 進捗管理、全完了時は自動アーカイブ | ✅ |
| OAuth認証 | Cloudflare Workers経由のSlack OAuth | ✅ |

## 推奨ワークフロー

```
1. get_slack_thread  → スレッド取得（文脈DB化）
2. analyze_request   → 目的・不明点・確認案を生成
3. draft_reply       → 返信の下書きを添削・構造化
4. save_task         → タスクとして保存
5. complete_step     → 進捗管理
```

## 設計原則

### ADHDフレンドリー

- 5分以内で終わるステップに分解
- 最初のステップは最も簡単なものに
- 途中で止めてもOKな区切りを明示
- 達成感を可視化（☑️チェックマーク）

### ワーキングメモリ節約

- スレッドの内容はタスクと紐づけて保存
- 「あの話どうなったっけ」をClaudeに聞ける
- スレッドを読み直す必要がない

### 構造化された返信

- 結論→根拠→アクションの形式
- 冗長な表現を簡潔に
- 論理の飛躍をチェック

## 成功指標（KPI候補）

1. メンション確認から返信までの時間短縮
2. タスク完了率の向上
3. 「何をすべきか迷う」時間の削減

## 非機能要件

- Claude Code / Claude Desktop との統合（MCP準拠）
- ローカルファイルでのデータ永続化（プライバシー重視）
- User Token使用（Botをチャンネルに追加不要）
