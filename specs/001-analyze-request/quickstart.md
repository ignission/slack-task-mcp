# Quickstart: analyze_request

**Feature**: 001-analyze-request
**Date**: 2025-12-18

## 概要

`analyze_request` は、Slackスレッドの依頼を分析し、以下を構造化して出力するMCPツールです：

- 目的の明確化
- 不明点の洗い出し
- 確認メッセージ案
- ネクストアクション

## 基本的な使い方

### 1. スレッドを取得して分析

```
ユーザー: このスレッドを分析して https://xxx.slack.com/archives/C12345678/p1234567890
```

Claude が自動的に:
1. `get_slack_thread` でスレッドを取得
2. 内容を分析
3. `analyze_request` で結果を構造化

### 2. 出力例

```
## 依頼の分析

### 把握した内容
- **目的**: 先週の進捗をまとめた週次レポートを作成し、チームに共有する
- **成果物**: 週次レポート（ドキュメント）
- **期限**: 金曜日 17:00
- **優先度**: 🟡 中（今週中にやるべき）

### 不明点
- ❓ **フォーマットは決まっていますか？**
  - 影響: フォーマットが異なると作り直しになる可能性がある
  - 想定される選択肢: 前回と同じ形式 / 新しいテンプレート / 自由形式

### 確認メッセージ案
「週次レポートについて確認させてください。フォーマットは前回と同じ形式でよろしいでしょうか？」

### ネクストアクション
📌 **山田さんにフォーマットについて確認メッセージを送る（2分）**
   理由: フォーマットが決まらないと作業を始められないため
```

## ユースケース

### ケース1: 曖昧な依頼

**入力**:
```
「来週のミーティングの資料作っておいて」
```

**出力**:
- 目的: ミーティング用の資料を作成する
- 不明点:
  - ミーティングの目的・アジェンダは？
  - 対象者は誰？
  - 必要なページ数・ボリュームは？
  - 期限（来週のいつ？）
- 確認メッセージ案: 「資料作成について確認させてください。ミーティングの目的とアジェンダを教えていただけますか？また、期限は来週の何曜日でしょうか？」

### ケース2: 明確な依頼

**入力**:
```
「12/20（金）15時までに、Q3の売上レポートをPPTで作成してください。対象は経営会議用で、5ページ程度でお願いします。」
```

**出力**:
- 目的: Q3の売上レポートを経営会議用に作成する
- 成果物: PowerPointスライド（5ページ程度）
- 期限: 12/20（金）15:00
- 不明点: なし
- ネクストアクション: 「Q3の売上データを収集する（5分）」

### ケース3: 複数の依頼が混在

**入力**:
```
「あ、そういえば先週の議事録まだだっけ？あと来週のミーティングのアジェンダも考えておいて」
```

**出力**:
- **依頼1**: 先週の議事録を作成する（優先度: 高 - 遅れている）
- **依頼2**: 来週のミーティングのアジェンダを作成する（優先度: 中）
- ネクストアクション: 「まず議事録から着手。Slackで先週のミーティングチャンネルを開く（2分）」

## ワークフロー全体

```
1. ユーザー: 「このスレッドを分析して [URL]」
   ↓
2. Claude: get_slack_thread(url) でスレッド取得
   ↓
3. Claude: 内容を分析（CLAUDE.mdのルールに従う）
   ↓
4. Claude: analyze_request(thread_content, analysis) で結果を構造化
   ↓
5. ユーザー: 「確認した。分解してタスクとして保存して」
   ↓
6. Claude: save_task(analysis, steps) でタスク保存
   ↓
7. ユーザー: 「ステップ1完了」
   ↓
8. Claude: complete_step(task_id, step_id) で進捗更新
```

## エラーケース

### 依頼が見つからない場合

```
## 分析結果

依頼が見つかりませんでした。

このスレッドには具体的な依頼・タスクが含まれていないようです。
以下の可能性があります：
- 雑談や情報共有のみのスレッド
- 既に完了した依頼についての会話
- 依頼者が別のスレッドで依頼している

別のスレッドURLを指定するか、依頼内容を直接入力してください。
```

## Tips

### 効果的な使い方

1. **まとめて分析**: 複数のメンションがある場合は、優先度順に整理される
2. **確認メッセージをコピペ**: 生成された確認メッセージはそのままSlackに貼り付けられる
3. **ネクストアクションから始める**: 分析後は提示されたネクストアクションからすぐ着手

### カスタマイズ

CLAUDE.md の「依頼分析のルール」を編集することで、分析のルールをカスタマイズできます。
